How I build courses — a 12-phase pipeline

A reusable, opinionated workflow for producing bilingual (EN/TH) self-paced training courses with wiki-backed citations, instructor slide decks, and a marketing landing page — deployable in 30 minutes.

Phase 01~30 min

Scope

Before writing anything — commit a scope.md with four answers.

  1. One-sentence outcome: "After this, the reader will be able to ____."
  2. Audience: Junior? Tech lead? Specific role?
  3. Format: Self-paced / 1-day workshop / course series
  4. Time budget: 30 min / 2 h / 8 h?

If you can't write all four in 30 minutes, the topic isn't ready. Don't proceed.

Output

scope.md committed

Tools

your favourite editor

Done when: scope.md exists with all four answers.
Phase 02~2-4 h

Research & capture

Find 3-5 primary sources — official docs, engineering blogs, formal specs.

Avoid "5 best ways to..." aggregator articles. You want the canonical source so your wiki claims have authority.

Save each source verbatim as raw/YYYY-MM-DD-<slug>.md with frontmatter:

---
source_type: web | doc | transcript
source_url: https://...
ingested_at: YYYY-MM-DD
title: <verbatim title>
---

Filter secrets before saving — replace tokens with <REDACTED:kind>.

Output

≥3 files in raw/

Tools

WebFetch · curl · pandoc

Done when: ≥3 raw files exist, each with valid frontmatter, captured verbatim.
Phase 03~30 min

Wiki init

Customise SCHEMA.md for your domain.

  • Entity types: add domain-specific (Kubernetes → resource, controller; Agent Skills → skill, concept).
  • Relations: keep the set small — uses, depends-on, composes, alternative-to, supersedes.
  • Half-life: shorten if the domain changes fast. JS frameworks: 90 days. Linux fundamentals: 365 days.

Output

Customised SCHEMA.md

Template

SCHEMA.md →

Done when: SCHEMA.md reflects your domain, not the generic template.
Phase 04~2-3 h

Ingest

Turn raw sources into wiki pages with citations.

For each entity in raw: decide new vs update. New entities → wiki/<type>:<slug>.md. Every claim cites raw + has confidence:

- <Claim.> `[src: raw/2026-05-08-source.md] {conf: 0.5}`

Reinforcement formula: conf ← 1 - (1 - conf) × 0.6

Update graph/entities.jsonl + graph/edges.jsonl as you go.

Rule of thumb: ~10-15 wiki pages per 12-module course.

Output

Cited wiki pages + graph

Tools

Claude with course-portal skill

Done when: every claim has citation + confidence, wiki-index.md lists every page.
Phase 05~1 h

Outline

Draft 10-12 modules. Every module must cite ≥1 wiki page.

If a module has nothing to cite, the domain isn't ready — go back to Phase 4.

Pattern that works:

  1. Welcome & what you'll learn
  2. Definition & analogy
  3. Anatomy / contract
  4. Core mechanism
  5. Ecosystem / where it fits
  6. Surfaces / variants
  7. vs alternatives (decision framework)
  8. Best practices
  9. Lab / hands-on
  10. Implementation guide
  11. Security & rollout
  12. Cheatsheet & resources

Output

outline.md with module → wiki page mapping

Done when: every module maps to at least one wiki page.
Phase 06~3-5 h

Course pages

Fill in course-en.html — long-form, single-file, sticky-sidebar TOC.

Each module structure:

  • Number + title + 1-sentence summary
  • 2-3 paragraphs of body
  • 1-2 code blocks or examples
  • 1 callout (Takeaway / Lab / Note)

Internal anchors (#m1, #m2...) so the sidebar TOC works.

Output

course-en.html

Theme

Dark/light toggle, Inter + JetBrains Mono

Done when: every anchor resolves, every wiki citation valid.
Phase 07~2-3 h

Slide deck

~30-35 reveal.js slides condensing the course.

Density rules I learned the hard way:

  • Max 6 bullets per slide
  • Max 15 lines of code per slide
  • Max 5 columns × 5 rows per table
  • Use canvas 1280 × 800 — not reveal.js's default 960 × 700. Default crops content.
Reveal.initialize({
  width: 1280, height: 800,
  margin: 0.06,
  minScale: 0.2, maxScale: 1.8,
  ...
});

Output

slides/training-en.html

Done when: no slide overflows the canvas when projected.
Phase 08~3-4 h · optional

Bilingual mirror

Mirror EN → TH (or skip for fast path).

Rules that work in production:

  • Code blocks stay in English — never translate identifiers.
  • Technical English terms preserved in <code> or <span class="term-en">.
  • Line-height 1.7 for Thai (1.65 for English) — Thai script has top/bottom marks.
  • Font: Noto Sans Thai first, then Inter.

Output

course-th.html + slides

Done when: TH structure mirrors EN, language flows naturally.
Phase 09~1-2 h

Landing page tuning

Polish index.html — marketing, not "training portal".

The 8 sections that work:

  1. Sticky nav + theme toggle + primary CTA
  2. Hero with gradient blur, animated badge, bilingual title, 2-3 CTAs
  3. Stats bar (4 numbers building credibility)
  4. Why (before/after 2-col)
  5. What you'll learn (3-col, 6 cards)
  6. Code/visual preview (real example)
  7. How it works (3-tier diagram of core mechanism)
  8. Get started → Final CTA → Footer

Output

Polished index.html

Done when: light + dark modes both polished, responsive at 375px.
Phase 10~30 min

Deploy

One-shot deploy to GitHub Pages.

git init -b main
echo ".DS_Store" > .gitignore
touch .nojekyll
git add . && git commit -m "Initial: <course>"
gh repo create your-org/<repo> --public --source=. --push
gh api -X POST /repos/your-org/<repo>/pages \
  -f "build_type=legacy" \
  -f "source[branch]=main" \
  -f "source[path]=/"
sleep 30
curl -I https://your-org.github.io/<repo>/

.nojekyll is critical — without it, Pages chokes on filenames containing colons (your wiki uses them).

Output

Live URL returning 200

Done when: all entry-point URLs return HTTP 200.
Phase 11~1-2 h

Validate

Friction tests with real readers, not your imagination.

  • Internal links: grep + curl every href
  • Mobile responsive: DevTools 375px width
  • Theme toggle: click sun/moon, scroll every section
  • Reading flow: hand to one peer + one junior, observe friction in the first 30 minutes
  • Citations: spot-check 5 quotes against wiki/raw

Output

Feedback notes + fix list

Done when: ≥3 rounds of feedback incorporated.
Phase 12~1 h

Promote

Make it findable.

  • Featured/Courses card on the personal hub
  • Blog post: "What I learned writing X" (~600 words)
  • Repo description + topics
  • Tweet / Medium

Output

Hub updated, draft post in repo

Done when: hub Featured/Courses card live, blog post drafted.
